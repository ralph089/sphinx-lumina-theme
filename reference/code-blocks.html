Basic Syntax Highlighting¶

Specify the language after the opening fence for syntax highlighting.

Python¶

def fibonacci(n: int) -> list[int]:
    """Generate the first n Fibonacci numbers."""
    sequence = []
    a, b = 0, 1
    for _ in range(n):
        sequence.append(a)
        a, b = b, a + b
    return sequence

Rust¶

fn binary_search<T: Ord>(slice: &[T], target: &T) -> Option<usize> {
    let mut low = 0;
    let mut high = slice.len();
    while low < high {
        let mid = low + (high - low) / 2;
        match slice[mid].cmp(target) {
            std::cmp::Ordering::Equal => return Some(mid),
            std::cmp::Ordering::Less => low = mid + 1,
            std::cmp::Ordering::Greater => high = mid,
        }
    }
    None
}

Line Numbers¶

Add :linenos: to display line numbers alongside the code.

 1class EventEmitter:
 2    def __init__(self):
 3        self._handlers = {}
 4
 5    def on(self, event: str, handler):
 6        self._handlers.setdefault(event, []).append(handler)
 7
 8    def emit(self, event: str, *args, **kwargs):
 9        for handler in self._handlers.get(event, []):
10            handler(*args, **kwargs)

The MyST syntax:

```{code-block} python
:linenos:
your code here
```

Including Code from Files¶

The literalinclude directive pulls code directly from a file in your project. This keeps examples in sync with real code.

docs/conf.py¶
"""Sphinx-Lumina-Theme documentation."""

from sphinx_lumina_theme import __version__

project = "Sphinx Lumina Theme"
extensions = [
    "myst_parser",
    "sphinx_design",

The MyST syntax:

```{literalinclude} ../path/to/file.py
:language: python
:caption: path/to/file.py
```

Useful Options¶

Option

Description

:language:

Syntax highlighting language. Auto-detected from file extension if omitted.

:lines:

Include only specific lines. Examples: 1-10, 5,8,12, 1-5,10-15.

:linenos:

Show line numbers.

:emphasize-lines:

Highlight specific lines within the included range.

:caption:

Add a caption above the code block.

:start-after:

Include content after a matching string (exclusive).

:end-before:

Include content before a matching string (exclusive).

Tip

Use :start-after: and :end-before: with comment markers for resilient includes that won’t break when line numbers change:

```{literalinclude} ../src/config.py
:start-after: # -- theme options start --
:end-before: # -- theme options end --
```

Copy Button¶

All code blocks include a copy-to-clipboard button provided by sphinx-copybutton. Hover over any code block to see it appear in the top-right corner. No configuration needed — it works automatically.
